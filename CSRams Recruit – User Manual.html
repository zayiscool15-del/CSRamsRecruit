Section 1

Overview of CSRams Recruit

CSRams Recruit is a web experience designed for the WSSU Computer Science & Information Technology Open House. It helps prospective students explore programs, register their interest, log in as students, and allows teachers to view and manage those registrations.

The site includes separate flows for students and teachers, all wrapped in a clean, WSSU-branded interface. This manual explains what each page does and how a visitor is expected to move through the system.

High-level flow: Explore programs → Register → Student Login & recommendations → Teacher Login → Teacher Dashboard.
Section 2

Site Navigation & Layout

A consistent navigation bar appears at the top of every page, making it easy to move between sections of the site.

Navigation items:

  • Home – WSSU CS Open House landing page
  • Programs – Computer Science & Information Technology programs
  • Register – WSSU Program Registration form
  • Student Login – Access to student dashboard & recommendations
  • Teacher Login – Access to teacher dashboard
  • Teacher Dashboard – Manage student registrations
How to use: Click any menu item once to switch pages. The navigation bar remains at the top so users can always move to another section.

The Register page is where prospective students formally submit their interest in WSSU Computer Science or Information Technology.

Section 6

Student Login – Interest & Recommended Courses

The Student Login page allows students to access their personalized WSSU CS & IT dashboard. From there, they can choose their interest area and receive recommended courses.

Page layout includes:

  • WSSU logo and branding
  • Heading: Student Login
  • Subheading: “Access your WSSU Computer Science & IT student dashboard.”
  • Email field
  • Password field
  • Login button (red)
  • Note: “Static demo accepts any entered email/password after registration.”

After logging in, students can:

  • Select their interest area (e.g., AI, cybersecurity, software engineering, IT, etc.)
  • View a list of recommended courses based on that interest
  • See their basic information and selections in a student dashboard view
This page is the bridge between registration and personalized guidanceStudent Flow—it turns a simple signup into a tailored course recommendation experience.

Section 8

Teacher Dashboard – Managing Student Registrations

The Teacher Dashboard is the administrative view where teachers can see all registered students and manage their records.

Page layout includes:

  • WSSU logo and branding
  • Heading: Teacher Dashboard
  • Subheading: “Manage student registrations. In this static version, data is stored only in the browser.”
  • A table with columns:
    • Student ID
    • Name
    • Email
    • Phone
    • Location
    • Program
    • Semester
  • If no students exist yet, the message: “No students registered yet.”
  • Logout button (red) at the bottom

Teacher actions:

  • View all students who have registered through the Register page
  • Optionally delete students from the list (if delete controls are implemented)
  • Log out and return to the Teacher Login page
This page represents the “admin view” of your projectTeacher Flow—it shows how teachers could manage student interest data in a real system.
Section 9

Demo & Data Storage Notes

CSRams Recruit is implemented as a static demo, which means it focuses on user experience and flow rather than full backend integration.

Key demo behaviors:

  • Student Login: Accepts any email/password combination after registration.
  • Teacher Login: Does not verify real credentials; any username/password works.
  • Teacher Dashboard: Stores data only in the browser (e.g., local storage) and does not persist across all environments.
Important: In a production system, all logins would be secured, and student data would be stored in a protected database. This project demonstrates the interface and flow, not full authentication.
